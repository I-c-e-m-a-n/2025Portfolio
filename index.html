Personal Project

Multi-Provider Web Search Service

Python · FastAPI Docker Compose Private repo, details on request
Python FastAPI Pydantic v2 httpx Docker Compose OpenAPI
Problem Each search provider has different schemas, rate limits, and failure modes. Callers should not need to know which provider is live.
Design Four-layer stack: API routing, service orchestration, provider adapters, result ranking. Providers are independently swappable; the interface is stable regardless of which backend answers.
Hard part Per-provider circuit breaker with sequential fallback, size-bounded streaming so oversized payloads cannot stall the pipeline, and SSRF guards on every redirect hop.
Outcome SearXNG, DuckDuckGo, Brave, and Serper unified behind one OpenAPI interface. Deployed in Docker Compose with a self-hosted SearXNG sidecar.
Artifacts OpenAPI schema (self-hosted at /docs) /v1/health per-provider endpoint Docker Compose stack Repo private, code walkthrough available on request
Architecture and implementation notes

Observability

Request-ID tracing middleware on every request. Structured JSON logging. Per-provider health at /v1/health. OpenAPI + ReDoc at /docs.

Resilience

Per-provider circuit breaker (closed, open, half-open). Exponential backoff with jitter. Semaphore-bounded concurrency. Stale-cache fallback on provider failure.

Security

SSRF guard: scheme whitelist, RFC1918 + loopback + cloud metadata IP block, DNS validation. Manual redirect handling with per-hop security check.

Personal Project

Multi-Service AI Backend with Capability Routing

Python · FastAPI · React 900+ tests Private repo, details on request
Python FastAPI React TypeScript SQLite Pydantic v2
Problem Chat, memory, task planning, and web retrieval need to run as separate services with independent failure domains, but still share a coherent request lifecycle.
Design Central router classifies each request into a typed intent and dispatches to the correct backend service. Each service declares a capability manifest; the router enforces those boundaries at the API layer.
Hard part Preventing older keyword-routing logic from overriding structured intent classification. Required a three-layer gate: intent compiler, execution planner, capability registry, in that order of authority.
Outcome Backend serving chat, memory retrieval, task orchestration, and web search. React and TypeScript frontend with server-sent event streaming. CLI and web share the same backend contract.
Artifacts 900+ unit & integration tests per-request trace records SSE streaming frontend Repo private, code walkthrough available on request
Architecture and implementation notes

Routing

Intent compiler classifies prompt into an IntentObject (mode, domain, capability, arguments). Execution planner builds step graph. Intent authority gate prevents legacy keyword routing from overriding structured intent.

Memory

Three-layer: episodic (per-session), semantic (entity/fact), procedural (learned patterns). Hybrid BM25 + embedding search. Sanitization on all write and injection paths.

Test coverage

900+ unit and integration tests. Regression suites gate all pipeline changes. Trace records per request for audit and debug.

Problem Children on the autism spectrum often cannot participate in group music activities because the soundscape is fixed and uncontrollable, leading to sensory overload.

Real-Time 3D Tactical Map with WebSocket Authority Server

Private repo, details on request
TypeScript React Three Fiber WebSocket Zustand Zod pnpm workspaces
Problem Multiple clients need to share a consistent 3D world state. The server must be the single source of truth, but clients still need to feel responsive.
Design Intent, validate, ACK and PATCH broadcast model over WebSocket. The server owns all mutations and broadcasts validated diffs. Shared Zod schemas across all four monorepo packages enforce the wire protocol.

Outcome Authoritative server maintaining a validated 3D voxel grid with terrain, fog-of-war, tokens, obstacles, and effects. React Three Fiber UI consuming live server diffs.
Artifacts shared/src/protocol.ts (Zod schemas) pnpm workspace: shared, mcp-map-server, mcp-world-ops-agent, ui Repo private, code walkthrough available on request

Protocol & State

Agent Bridge

How I work

Retrieval accuracy is measurable. Getting to 12–15 ms at ~90% hit rate requires scoring pipelines that are deterministic and tunable at every stage, not end-to-end black boxes.

Evaluation comes before shipping. If a ranking or scoring change cannot be verified against a regression suite, it has not been validated.

Building for people with strict sensory constraints removes the option to hand-wave over edge cases. Every hard constraint clarifies what the system actually needs to do.
