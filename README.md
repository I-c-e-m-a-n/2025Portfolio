## Stack

| Layer | Technology |
|---|---|
| Markup | Semantic HTML5 — no framework, no templating |
| Styling | `assets/style.css` — custom CSS design system (tokens, components, dark mode) |
| Icons | Lucide `v0.475.0` — self-hosted UMD at `assets/lucide.min.js`, pinned build |
| Fonts | Google Fonts — Inter · JetBrains Mono — non-render-blocking (`media=print` swap) |
| Behaviour | `assets/main.js` — vanilla JS, zero runtime dependencies, loaded with `defer` |
| Hosting | GitHub Pages — static, no build step, deployed via GitHub Actions |

No Tailwind. No React. No bundler. No runtime CSS generation.

## Features

- **Dark/light mode** — persisted to `localStorage`, applied inline before first paint (no flash); `no-js` CSS fallback
- **Scroll progress bar** — 1.5px gradient indicator pinned to nav bottom
- **Active section tracking** — `IntersectionObserver` with intersection-ratio map; sets `aria-current` and `--section-accent` CSS variable
- **Scroll reveal** — `translateY` fade-in on viewport entry; auto-stagger by sibling group; skipped for `prefers-reduced-motion`
- **Mobile nav drawer** — slide-in with full focus trap, `Escape` to close, `aria-modal`, focus return to trigger element
- **Expandable tech detail panels** — native `<details>`/`<summary>`; icons and reveal items activated on open
- **Proof strips and artifact rows** — truthful external links or explicit private-repo labels
- **Inline SVG architecture diagrams** — `currentColor` tokens work in both light and dark mode
- **Accessibility** — skip link, `aria-modal`, `aria-expanded`, `aria-current`, `aria-label` on all landmark regions, `focus-visible` on all interactive surfaces
- **SEO** — `<title>`, `description`, canonical, OG, Twitter Card (`summary_large_image` with rasterized PNG), JSON-LD `Person` schema, SVG favicon, `color-scheme` meta

## Local Development

No build step. Serve with any static file server:

```bash
# Python (built-in)
python3 -m http.server 8080
# open http://localhost:8080

# Node
npx serve .
```

`og-image.png` is only generated in CI. Locally the OG image will 404 — this is expected and does not affect any visible functionality.

## Deployment

Push to `main`. The `deploy.yml` workflow:
1. Converts `assets/og-image.svg` → `assets/og-image.png` using `librsvg` (Twitter/X requires rasterized PNG for `summary_large_image`)
2. Uploads the repo as a Pages artifact
3. Deploys via `actions/deploy-pages`

**Prerequisite:** In the repo Settings → Pages → Source, select **GitHub Actions** (not the legacy `gh-pages` branch).

## Updating the Lucide Icon Set

The icon file is pinned at `v0.475.0`. To upgrade:

```bash
curl -L -o assets/lucide.min.js \
  "https://unpkg.com/lucide@<NEW_VERSION>/dist/umd/lucide.min.js"
```

Update the version comment in `index.html` line 66 and this README after replacing the file.

## Design Principles

- **No runtime dependencies** — all CSS is static, all JS is vanilla with `defer`
- **Typography-led** — Inter for display and body; JetBrains Mono for code, dates, and metadata chips
- **CSS custom properties throughout** — dark mode via `.dark` class toggling `:root` overrides; `no-js` fallback class removed by inline script
- **Truthful content** — all metrics, claims, and project details are traceable to the resume or verified local repositories
- **Accessibility is structural** — focus states, focus traps, skip links, and `aria-*` attributes are deliberate, not decorative
